ZeroClaw is the runtime operating system for agentic workflows — a single Rust binary that abstracts AI models, communication channels, tools, memory, and execution so you can build agents once and run them anywhere. It starts in under 10ms, uses less than 5MB RAM, and runs on everything from a $10 board to a cloud VM.

## Why ZeroClaw

ZeroClaw is built for teams and individuals who need an autonomous AI assistant that is lean, secure, and fully under their control.

<CardGroup cols={2}>
  <Card title="Lean by default" icon="feather">
    Less than 5MB RAM and sub-10ms cold starts. Runs on a $10 ARM board with the same binary as your production server.
  </Card>
  <Card title="Secure by design" icon="shield-halved">
    Gateway pairing, strict sandbox, filesystem scoping, encrypted secrets, and deny-by-default channel allowlists.
  </Card>
  <Card title="Fully swappable" icon="arrows-rotate">
    Every subsystem is a trait — swap AI providers, messaging channels, memory backends, and tools with a single config line.
  </Card>
  <Card title="No lock-in" icon="unlock">
    OpenAI-compatible and Anthropic-compatible provider support. Run any model locally with Ollama, llama.cpp, or vLLM.
  </Card>
</CardGroup>

## Get started in three steps

<Steps>
  <Step title="Install ZeroClaw">
    Install via Homebrew, the one-line script, or download a pre-built binary for your platform.

    ```bash
    # macOS / Linuxbrew
    brew install zeroclaw

    # One-line installer
    curl -fsSL https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/install.sh | bash
    ```
  </Step>
  <Step title="Run onboarding">
    Configure your AI provider, channels, and workspace in a single command.

    ```bash
    zeroclaw onboard --api-key sk-... --provider openrouter
    ```
  </Step>
  <Step title="Chat with your agent">
    Send a one-shot message or launch an interactive session.

    ```bash
    # Single message
    zeroclaw agent -m "Summarize the files in my workspace"

    # Interactive mode
    zeroclaw agent
    ```
  </Step>
</Steps>
